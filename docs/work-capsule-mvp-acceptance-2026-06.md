# Work Capsule MVP Acceptance Review

June 2026

## Product Promise

AI Chat Vault provides local reusable work memory from selected ChatGPT turns. A user can choose useful turns from a captured ChatGPT conversation, shape them into an editable Work Capsule, save it locally, and reuse the resulting context without rereading the whole chat.

## Privacy Boundary

The default Work Capsule loop remains local-first. Current evidence preserves this boundary:

- `public/manifest.json` keeps permissions to `storage` and host access to `https://chatgpt.com/*` and `https://chat.openai.com/*`.
- `src/workCapsules.ts` stores records through `chrome.storage.local` keys `workCapsules:v1` and `workCapsule:v1:<id>`.
- `src/workCapsules.ts` renders bridge presets as local text only; it does not call provider APIs.
- `tests/workCapsules.test.ts` and `tests/popup.test.ts` include guards that stub `fetch` and `XMLHttpRequest` and assert they are not called.
- `README.md` and `docs/work-capsule-bridge-presets.md` state that bridge presets are manual, local text renderings and do not add backend, analytics, cloud sync, remote LLM calls, provider-side memory updates, provider UI automation, uploads, or broader Chrome permissions.

## Next Earliest Useful Iteration

Add a local "Context Prompt only" copy preset or adjacent action that copies just the editable `contextPrompt` field. This stays inside the Work Capsule MVP, clarifies the difference between concise prompt reuse and fuller bridge-preset handoff, and requires no backend, analytics, cloud sync, remote LLM call, provider automation, or broader Chrome permissions.
